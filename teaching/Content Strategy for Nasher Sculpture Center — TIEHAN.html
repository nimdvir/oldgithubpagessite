Summary

 

About The client:
Nasher sculpture center

Nasher Sculpture Center is a museum that collects and exhibits contemporary sculptures in Dallas, Texas. It is very reputable in the local community and they are called jewel of Dallas. It owns a beautiful garden and provides venue reservation for all the guests. This museum has an annual Nasher Prize for global sculpture artists.

Website Introduction: https://www.nashersculpturecenter.org/visit/about-the-nasher

 

Project Direction

  • Audit all Nasher’s digital content

  • Enhance Nasher’s Search Engine Optimization

  • Create better online membership purchasing experience

  • Design new prototype for Nasher’s website

Client Objectives

  • Drive more online revenue through membership purchases/renewals and private event reservations.

  • More online engagement by exploring deeper into their website

  • Building more user’s loyalty ultimately encouraging them to become visitors and members

Solutions

 

Pain point 1: SEO

SEO errors in WooRank evaluation result

Search results for “Nasher”

 

Based on my SEO comparison research, we can see Nasher’s SEO performance is not that well.

If we search for “Nasher” by Google, the top result will be Nasher Museum of Art at Duke University.

 

Recommendation for SEO

Nasher SEO Checklist

 

Optimize the website’s tags and descriptions

  1. Follow the Checklist and solve all technical problems that may infect the SEO performance.

  2. Add more “Nasher” as keywords into tags/Alts/Descriptions to increase the exposure rate on Google

 

Pain point 2: Information Architecture

Nasher’s homepage navigation samples

  • There is no landing page for each independent navigation part. (User often click the navigation items but no response happens)

  • Put different information hierarchy name in a same category. (Intuitively, Artist Microgrants should under Programs)

  • Some titles of the menu are not very relevant with submenu title (Art and French sculpture census)

  • Some names of submenu are vague (The Great Create)

From reviewing the content audit, I found some pages can be integrated as one page because their content can basically be categorized in a same page

 

Recommendations for IA

The new Recommended IA

Principles of new Information Architecture

  1. Simplification: Don’t let the Navigation items more than 5 for better memory.

  2. Clear and understandable label name: Do not use repetitive or similar label name to make the user feel confused.

  3. Create landing pages for each navigation label: Gather all the overview information into these landing pages so that user can directly go the page that shows more detailed information that user wants.

 

Example of Art landing page

 

Design Statement

Design landing page for all of the navigation items. These pages will have a subordinate navigation to show the items under it and help user navigate to the part they want.

 

Pain point 3: Weak first impression

Comparison with the homepage of competitors

Homepages of Nasher and its competitors, Nasher’s sense of brand image and the its uniqueness are relatively weak.

 

Recommendations for first impression

New Layout for Nasher’s homepage

 

Design statement

Attempt to use more unique layout to show all the Nasher’s uniqueness and leave stronger impression to users. Also repeat the theme color visually and create more impressive branding feelings.

Concerns

The first layout may cause controversial evaluations from different users, so I also made a relatively conservative design layout for the following “upcoming events” part, as a backup plan.

 
 

Emphasize the location information

Based on the user research, most of users are concerned about the location of the museum. Not only showing the text address information, I add location information by showing the map and the button to show directions (I will navigate user to the Google Map).

The map part on the homepage

 

Pain point 4: Online purchasing experience

The first page after clicking “become a membership”

 

Membership purchasing page displays information that user doesn’t expect. Based on my user interviews, the user expects the price info and the benefit bullet points.

 

Recommendations for Online purchasing experience

New designed membership page

 

Design Statement

Not like the original website, having tedious processes before showing the real membership information, user can just do one click to view to all the membership information including all the plans, prices, and benefits.

  • Optimizing the page layout for the whole online purchasing system

  • Emphasize useful information on the most obvious place. (price, benefit, How to be a membership…)

 

Pain point 5: Lack of advantages exposure

The first sight of Nasher’’s homepage

 

The homepage doesn’t showcase Nasher’s uniqueness and advantages:

  • The garden with natural environment

  • Their rich collections

 

Recommendations for showcasing advantages

The first sight of my prototype

Design Statement

Make carousel album as the first information. Its images keep rolling and show different images at the same time. Show the most popular and visually appealing collections as cover images.

Showcase the garden and the beautiful scenes of the museum on the homepage.

Use VR tour to let user have more immersive experience to know the beauty of Nasher garden.

 

Little Tags

Use little tags to show some features of the events.

Also emphasize the benefits of the membership.

 

Little tag for each event

 
 
Design Prototype

Future steps

Questions need to be answered: Is the structure making sense to user? How is the user’s feeling the membership purchasing experience? How effective this website motivate user to become a member. Will user want to explore more on the website?

Iterations based on feedbacks

Make some language adjustments based on the user feedback.

Create another journey map for content consumer and not about planning a visit. Adjust the prototype for them!
